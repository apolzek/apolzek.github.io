## Cheatsheet openssh

```sh
# Gerar um novo par de chaves SSH (algoritmo RSA, 4096 bits)
ssh-keygen -t rsa -b 4096

# Copiar a chave pública para um servidor remoto (para autenticação sem senha)
ssh-copy-id usuario@servidor

# Conectar-se a um servidor remoto via SSH
ssh usuario@servidor

# Executar um comando em um servidor remoto e sair
ssh usuario@servidor 'comando'

# Configurar um túnel SSH (Port Forwarding) - encaminhar porta local para remota
ssh -L porta_local:servidor_intermediário:porta_remota usuario@servidor_ssh

# Encaminhar uma porta remota para uma local (Reverse Port Forwarding)
ssh -R porta_remota:localhost:porta_local usuario@servidor_ssh

# Usar um arquivo de chave privada específico para a conexão SSH
ssh -i /caminho/para/chave_privada usuario@servidor

# Conectar-se a um servidor utilizando uma porta específica
ssh -p porta usuario@servidor

# Habilitar o modo verbose para depuração de conexões SSH
ssh -v usuario@servidor

# Utilizar multiplexação de conexões para reutilizar conexões SSH existentes
# Primeiro, adicionar ao arquivo de configuração do SSH (~/.ssh/config):
Host *
  ControlMaster auto
  ControlPath ~/.ssh/sockets/%r@%h-%p
  ControlPersist 600
  
# Copiar arquivos de/para um servidor usando SCP
scp arquivo_local usuario@servidor:/caminho/destino
scp usuario@servidor:/caminho/remoto/arquivo_local /caminho/destino/local

# Copiar uma pasta inteira de/para um servidor usando SCP com opção -r
scp -r pasta_local usuario@servidor:/caminho/destino
scp -r usuario@servidor:/caminho/remoto/pasta_local /caminho/destino/local

# Usar SFTP para transferir arquivos com um servidor
sftp usuario@servidor

# Lista branca de IPs (permitir somente IPs específicos no servidor SSH - editar /etc/hosts.allow)
sshd : IP_1 IP_2 : allow

# Bloquear IPs específicos (editar /etc/hosts.deny)
sshd : IP_1 IP_2 : deny

# Alterar a porta padrão do SSH (editar /etc/ssh/sshd_config e mudar a porta)
Port 2222

# Desabilitar login SSH como root (editar /etc/ssh/sshd_config)
PermitRootLogin no

# Limitar o tempo de vida de uma sessão SSH inativa (editar /etc/ssh/sshd_config)
ClientAliveInterval 300
ClientAliveCountMax 0

# Forçar a utilização de autenticação por chave (editar /etc/ssh/sshd_config)
PasswordAuthentication no

# Recarregar a configuração do SSH sem reiniciar o serviço
sudo systemctl reload sshd

# Verificar a configuração do SSH para erros
sshd -t

# Verificar quem está conectado via SSH
who

# Encerrar uma sessão SSH específica
pkill -9 -t pts/N
```
